7/18

Today, I was given a more defined and elaborate lecture about functions in C. Functions must be declared before/above the main function. When programming in C, the functions have three parts: the declaration, the definition, and the call. The declaration of a function has a return type and a name and can be written with one of two varitations: types of parameters and parameters with their respective types. Calling a function requires providing values to the parameters. These space for parameters and the return value are added to the stack. When the function is completed, the local variables are unaccessible and the stack pointer goes to the return value.

7/19

Today in the lab, I experimented with the LEDs on the roach. In the assigned Lab, one of the tasks was to make the LED bar responsive to the amount of light the photoresistor is receiving. Using the given function "Roach_LightLevel()", I was able to scale the light sensor to return a value between 0 and 12. In the main function, we set the LEDs to respond to the amount of light by writing "Roach_LEDSSet(0b111111111111 << scaledLight)". This turns the led bar completely on and shifts it left based on the value returned by the scaled light function.
